Responsive Rules

Desktop-first with max-width queries. Exactly three breakpoints — never invent a fourth. Reference values live in tokens/layout.css.

Desktop
default
Full layout. Sidebars expanded, multi-column grids, hover states.
Tablet
≤ 1024px
Dashboard sidebar collapses to icon rail. 3-col grids drop to 2.
Mobile
≤ 768px
The primary collapse point: everything single-column, split heroes stack, brochure call bar appears.
Phone
≤ 640px
Tighten page padding to 20px; full-width buttons in forms and CTAs.
  • Hit targets ≥ 44px on mobile — buttons, nav rows, form controls.
  • Tables don't shrink — below 768px let them scroll horizontally inside their card, or switch to a stacked list.
  • Type doesn't shrink below the scale — display sizes may step down one scale stop on mobile; body never goes under 14px.
  • Spacing steps down one stop — section padding 96 → 64, card padding 24 → 20. Stay on the 4/8 scale.
  • No hover-only affordances — anything revealed on hover must also be reachable by tap or always visible on mobile.
  • One primary action stays visible — on mobile the view's solid button never scrolls out of the first screen of a form or task.
